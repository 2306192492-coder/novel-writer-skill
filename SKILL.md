# 📖 Novel Writer - 工业级长篇小说创作引擎 (v3.1)

## 🎯 核心升级 (v3.1)
- ✅ **向量检索引擎**: 内置轻量级语义搜索，写第 100 章也能秒找第 1 章的细节
- ✅ **强制暂停机制**: 每 50 章自动锁死，强制用户确认后续方向，防止失控
- ✅ **智能上下文加载**: 根据当前剧情，自动检索相关章节/设定，突破模型上下文限制
- (v3.0 原有功能保持不变：自动记忆/防崩坏/批量操作/多格式导出)

## 🚀 新增功能详解

### 1. 向量检索引擎 (解决长文记忆)

**问题**: 写到第 100 章时，模型记不住第 5 章的细节（上下文超限）。

**解决方案**:
- 每章生成后，自动切片存入**本地向量索引** (`~/.novel_index/`)
- 写新章节时，自动搜索相关段落（如"主角第一次突破筑基期"）
- 只加载**最相关的 5-10 个片段**，而非全文，节省 90% Token

**使用示例**:
```text
你：写第 100 章，主角再次突破
我：(自动检索)
  → 搜索"主角 突破 筑基期" → 找到第 5 章、第 23 章、第 67 章
  → 加载这 3 章的相关段落（共 800 字）
  → 生成第 100 章，确保境界设定一致
```

**技术实现**:
- 算法：TF-IDF + 余弦相似度（无需外部 API，纯本地）
- 分词：中文 2 字组合 + 单字
- 索引文件：`~/.novel_index/书名.json`

### 2. 强制暂停机制 (防止失控)

**问题**: 用户完全不管，AI 可能写到后期剧情崩坏。

**解决方案**:
- 每生成**50 章**，自动锁死，停止生成
- 输出**中期报告**（人物变化/伏笔回收率/战力评估）
- 强制用户确认："是否继续按当前方向写？还是要调整？"

**流程**:
```text
我：✅ 第 50 章生成完毕

🔒 【强制暂停】
已生成 50 章 (12.5 万字)，请确认后续方向：

【当前状态】
- 主角：筑基后期 (预期：第 100 章突破金丹)
- 伏笔：15 个已埋，8 个已回收 (回收率 53%)
- 战力：正常 (越级挑战 2 次，合理)

【建议】
1. 回收 3 个早期伏笔（神秘玉佩/失踪的师父/秘境钥匙）
2. 准备进入第二卷（换地图）

【操作】
- "继续" → 按当前方向写第 51-100 章
- "调整" → 修改大纲/人设/方向
- "查看报告" → 查看详细分析报告
```

**解锁方式**:
- 用户回复"继续" → 解锁，继续生成
- 用户回复"调整" → 进入修改模式
- 不回复 → 永久锁死（防止 AI 自说自话）

## 🧠 更新后的工作流

### 章节生成 (增强版)
**用户**: "写第 100 章"
**技能自动执行**:
1. **检查强制暂停**:
   - 如果当前是第 51 章且未确认 → 🔒 锁死，要求用户确认
   - 如果已确认 → 继续
2. **向量检索**:
   - 搜索"第 100 章相关设定"（如"金丹突破"）
   - 加载 Top 5 相关片段（第 5 章、第 23 章、第 67 章等）
3. **加载上下文**:
   - 前 3 章全文
   - 检索到的相关片段
   - 人物卡/伏笔表/时间线
4. **生成正文** → 2100-2500 字
5. **更新向量索引** → 将第 100 章切片存入
6. **检查强制暂停点**:
   - 如果刚写完第 50/100/150 章 → 🔒 锁死，输出报告
7. **输出** → 正文 + 自检报告

## 🔧 技术实现细节

### 1. 向量索引结构
```json
{
  "documents": {
    "第 01 章": "正文内容...",
    "第 02 章": "正文内容...",
    "人物卡": "主角：张三 | 性格：谨慎...",
    "伏笔表": "1. 神秘玉佩 (第 3 章首次提及)..."
  },
  "metadata": {
    "第 01 章": { "updatedAt": 1234567890, "wordCount": 2500 }
  }
}
```

### 2. 检索算法
```javascript
// 伪代码
function search(novelId, query, topK) {
  const documents = loadIndex(novelId);
  const scores = computeTFIDF(documents, query);
  return topK(documents, scores);
}
```

if (chapterNumber % 50 === 0 && !userConfirmed) {
  lock();

## ⚠️ 注意事项 (更新)

1. **向量检索精度**: 当前使用 TF-IDF，适合关键词匹配。如需语义理解，可升级到 Embedding 模型。
2. **强制暂停**: 如需连续生成（如冲榜期间），可临时关闭：`/set pause_interval 999`
3. **索引清理**: 删除小说时，手动删除 `~/.novel_index/书名.json`
